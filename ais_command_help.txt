## ais_start

Usage: ais_start [--help] [--version] [--verbose] [--show-log] [--show-splash] [--no-separator] [--no-open] [--start_with <corpus>]

Start all AIStudio services and open the UI.
Stops any running services first — safe to run even if already running.
Services: Qdrant (vector store), Ollama (LLM host), FastAPI backend.
On first run, automatically indexes the demo and help corpora (tracked in git;
Qdrant collections are empty until indexed). Backend health is polled every second for up to
30s — browser only opens once the backend is confirmed healthy.

Options:
  --help               Show this help
  --version            Show script version
  --verbose            Show full service output (Qdrant, uvicorn logs)
  --show-log           Open new iTerm2 tab with live backend log after start
  --show-splash        Show splash dialog on startup (useful when launching from Terminal)
  --no-separator       Use blank lines between sections instead of --- labels
  --no-open            Skip opening the browser (used by ais_create_shortcut desktop app)
  --start_with <name>  Open UI with the named corpus pre-selected
                       Example: ais_start --start_with sec_10k

· To stop:    ais_stop
· Logs:       ais_log
· To restart: ais_start

## ais_bench

Usage: ais_bench [--help] [--version] [--corpus <name>] [--top-k <n>] [--temperature <t>] [--model <name>] [--alpha <f>] [--firm <name>] [--subset <group>] [--topics <list>] [--question-ids <list>] [--questions <path>] [--api <url>] [--no-markdown] [--full]

Run the AIStudio benchmark suite against a corpus.
Default: demo corpus, top-k 5, temperature 0.3.

Options:
  --help                 Show this help
  --version              Show script version
  --corpus <name>        Corpus to benchmark (default: demo)
  --top-k <n>            Number of chunks to retrieve (default: 5)
  --temperature <t>      LLM temperature (default: 0.3)
  --model <name>         LLM model to use (default: from API config)
  --firm <name>          Inject firm filter into all queries (overrides YAML firm fields)
  --subset <group>       Run only questions whose firm is in the named subset
  --topics <list>        Comma-separated list of topic: values to include
  --question-ids <list>  Comma-separated list of question id: values to include
  --alpha <f>            Hybrid retrieval alpha: 0.0=pure vector, 1.0=pure BM25 (default: backend default)
  --questions <path>     Path to YAML question file, or bare stem without extension
                         (e.g. sec_10k_questions_no_filter auto-expands to
                         benchmarks/<corpus>/sec_10k_questions_no_filter.yaml).
                         Default: auto-detect from corpus name
  --api <url>            API base URL (default: http://localhost:8000)
  --no-markdown          Skip writing the .md report
  --full                 Include full answers in report (default: first 4 paragraphs)

Question subset filtering (sec_10k):
  ais_bench --corpus sec_10k --subset big_banks
  ais_bench --corpus sec_10k --topics "AI Risk Evolution,Cybersecurity"
  ais_bench --corpus sec_10k --question-ids capital_ratios_trend,latency_test
  ais_bench --corpus sec_10k --topics "Capital & Financial Position" --subset big_banks

· Subsets compose with --topics and --question-ids using AND
· Available subsets: big_banks, bulge_bracket, asset_managers, exchanges, insurance, custody, boutiques
· Requires: ais_start must be running
· Questions: benchmarks/<corpus>/<corpus>_questions.yaml
· Reports:   benchmarks/<corpus>/reports/ (filename includes filter suffix when active)

## ais_import_knowledge_base
ais_import_knowledge_base v1.0.0 — Import external knowledge source data for a corpus scope

Fetches entity or glossary data from an external knowledge source (K) for a specific
corpus (C) and scope (S), and writes the enriched output to data/knowledge_sources/<K>/.
Updates catalog.yaml with import metadata (date, count, file path).

Usage: ais_import_knowledge_base --source <K> --corpus <C> --scope <S> [options]
       ais_import_knowledge_base --list

Options:
  --source <K>      Knowledge source: gleif | sec_edgar | bis_basel | nist_ai_rmf | esrs
  --corpus <C>      Corpus name (e.g. sec_10k, esef_banks, any_corpus)
  --scope <S>       Scope ID matching a <C>_<S>_scope.yaml file (e.g. 25_firms, full)
  --force           Overwrite existing output file
  --list            Show all imported knowledge sources from catalog.yaml
  --help            Show this help
  --version         Show version

Examples:
  ais_import_knowledge_base --source gleif --corpus sec_10k --scope 25_firms
  ais_import_knowledge_base --source bis_basel --corpus any_corpus --scope full
  ais_import_knowledge_base --list

· Scope file:   benchmarks/<C>/<C>_<S>_scope.yaml (flat list of entity/term names)
· Metadata:     data/knowledge_sources/<K>/<K>_metadata.yaml (API config, license)
· Output:       data/knowledge_sources/<K>/<K>_<C>_<S>_<A>.yaml (A = entities|glossary|companies)
· Catalog:      data/knowledge_sources/catalog.yaml (registry of all imports)
· Sources implemented: gleif
· Sources planned:     sec_edgar, bis_basel, nist_ai_rmf, esrs

## ais_create_shortcut

Usage: ais_create_shortcut [--help] [--version] [--nodesktop] [--remove]

Create an AIStudio app shortcut on macOS.
Builds AIStudio.app, installs a Launch Agent, and adds a Desktop symlink.
Double-clicking the icon starts AIStudio silently — no Terminal needed.
Shows a splash notification while services start (~6 seconds).

Options:
  --help        Show this help
  --version     Show script version
  --nodesktop   Skip Desktop symlink (app in ~/Applications only)
  --remove      Remove AIStudio shortcut, app, and Launch Agent (full undo)

· Optional post-install step — run once after ais_install
· macOS only
· Launch log: ~/Library/Logs/AIStudio/launcher.log
· Docs: see docs/README - AIStudio - ais_create_shortcut.md
